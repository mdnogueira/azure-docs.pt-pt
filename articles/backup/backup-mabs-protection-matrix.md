## <a name="cluster-support"></a>Suporte de cluster
Servidor de cópia de segurança do Azure pode proteger dados nas seguintes aplicações em cluster:

-   Servidores de ficheiros

-   SQL Server

-   Hyper-V - se proteger um cluster de Hyper-V utilizando a proteção do DPM de escalamento horizontal, não é possível adicionar proteção secundária de cargas de trabalho protegidas do Hyper-V.

    Se executar o Hyper-V no Windows Server 2008 R2, certifique-se de que instala a atualização descrita no artigo KB [975354](https://support.microsoft.com/en-us/kb/975354).
    Se executar o Hyper-V no Windows Server 2008 R2 numa configuração de cluster, certifique-se de que instala SP2 e KB [971394](https://support.microsoft.com/en-us/kb/971394).

-   Exchange Server - servidor de cópia de segurança do Azure pode proteger clusters de disco não partilhados para versões do Exchange Server suportadas (replicação contínua de cluster) e também pode proteger o Exchange Server configurado para replicação contínua local.

-   SQL Server - servidor de cópia de segurança do Azure não suporta a cópia de segurança de bases de dados do SQL Server alojadas em volumes partilhados de cluster (CSVs).

Servidor de cópia de segurança do Azure pode proteger cargas de trabalho de cluster que estão localizadas no mesmo domínio que o servidor do DPM e num domínio subordinado ou fidedigno. Se pretender proteger origens de dados em grupos de trabalho ou domínios não fidedignos, utilize NTLM ou autenticação de certificados para um único servidor ou a autenticação de certificados apenas para um cluster.
